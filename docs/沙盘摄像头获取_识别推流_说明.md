## OpenCV 拉流示例

核心代码是：

```python
import cv2

rtsp_url = "rtsp://10.126.59.120:8554/live/live1"

cap = cv2.VideoCapture(rtsp_url, cv2.CAP_FFMPEG)
cap.set(cv2.CAP_PROP_BUFFERSIZE, 1)

if not cap.isOpened():
    print(f"无法打开视频流: {rtsp_url}")
    raise SystemExit

while True:
    ret, frame = cap.read()
    if not ret:
        print("读取帧失败，流可能已断开")
        break

    cv2.imshow("RTSP Stream", frame)
    key = cv2.waitKey(1) & 0xFF
    if key == ord("q"):
        break
    if key == ord("s"):
        cv2.imwrite("snapshot.jpg", frame)

cap.release()
cv2.destroyAllWindows()
```

## 识别后显示

文档用 YOLO 举例，流程是：

1. 打开 RTSP 视频流。
2. 对每一帧调用模型推理。
3. 把识别框和标签画回原图。
4. 用 `cv2.imshow()` 显示识别后的画面。

核心逻辑：

```python
from ultralytics import YOLO

model = YOLO("yolov8n.pt")
results = model(frame, verbose=False)
annotated = results[0].plot()
cv2.imshow("YOLO Detection", annotated)
```

在我们的项目里，YOLO 可以替换成自己的车牌识别、交警手势识别、车主手势识别等模型。

## 实时识别加推流

如果要把识别后的画面变成一个新的视频流，需要两个工具：

| 工具 | 作用 |
| --- | --- |
| mediamtx | 本地流媒体服务器，负责接收并发布视频流 |
| ffmpeg | 推流工具，把程序生成的帧编码并推送到 mediamtx |

文档给出的下载地址：

- mediamtx: `https://github.com/bluenviron/mediamtx`
- ffmpeg: `https://github.com/BtbN/FFmpeg-Builds/releases`

基本流程：

1. 启动 `mediamtx.exe`，终端不要关闭。
2. Python 程序从 `SRC_URL` 拉取沙盘摄像头流。
3. Python 程序对每帧做识别并绘制结果。
4. Python 把绘制后的帧写入 ffmpeg 的 stdin。
5. ffmpeg 推送到 `DST_URL`。
6. 浏览器打开 mediamtx 的 WebRTC 地址观看结果。

示例地址：

```python
SRC_URL = "rtsp://10.126.59.120:8554/live/live1"
DST_URL = "rtsp://127.0.0.1:8554/video30"
```

浏览器查看地址：

```text
http://127.0.0.1:8889/video30
```

## 和本项目的关系

本项目的“连接视频流”功能可以直接使用这些 RTSP 地址，例如：

```text
rtsp://10.126.59.120:8554/live/live1
```

前提是当前电脑能访问 `10.126.59.120` 所在网络。

如果只是在网页里实时识别，不一定需要 mediamtx 和 ffmpeg；前端/后端可以直接拉流或传帧识别。

如果想把“识别后的画面”再变成一个新的 RTSP/WebRTC 视频源，才需要 mediamtx + ffmpeg。
