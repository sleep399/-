# 小学期任务清单_2026

> 来源文件: `小学期任务清单_2026.pdf`

## 第 1 页

总体目标:开发一个前/后端分离的 Web 系统,对前/后端框架及开发语言不做
限制。本项目聚焦车载摄像头视觉感知与人机交互场景:利用车载前视/环视摄
像头实现道路车辆车牌识别、交警手势识别、车主手势控车。同时,引入智能
体(Agent)技术实现系统监控与自动告警,建议引入预训练模型、大语言模型
(Large Language Model,LLM)API 调用、提示词工程等技术辅助系统构建与
交互设计,不建议从零开始训练模型。
一、功能需求清单及技术要求
1. 道路车辆车牌识别(20 分)
基于车载前视/环视摄像头捕获的道路场景图像或视频流,实现对周围行驶
或停放的其他车辆(前车、旁车、对向车辆等)的车牌检测与字符识别。系统
需能够从复杂道路场景中自动检测并定位车辆的车牌区域,准确识别车牌号码
及颜色,为辅助驾驶、违章举报、停车场管理等场景提供感知能力。
(1) 支持行车记录仪/车载摄像头拍摄的道路场景图片上传与视频流输入
两种方式(3 分);
(2) 具备车牌检测与 OCR 识别完整流水线,对每辆检测到的车辆输出车
牌号码、车牌颜色(蓝牌/绿牌/等)等结构化信息(5 分);
(3) 实现识别结果的 Web 端实时展示,包括道路场景图上所有检测车辆
的车牌定位框标注与识别文字叠加显示,支持历史识别记录的存储与查
询(4 分);
(4) 模型能够在智慧交通管理系统场景化沙盘中运行并有效识别车牌信息
(可选,5 分);
(5) 能够在雨雪雾等复杂场景中正确检测并识别车牌信息(可选,3 分)。
2. 交警手势识别(20 分)
交警手势是道路交通管理中的重要信号。设计一个能够识别标准交警指挥
手势的视觉系统,基于视频或连续图像帧输入,实时识别交警的身体姿态与手
势动作,并将其映射为对应的交通指挥信号(停止、直行、左转弯、右转弯、
减速慢行、靠边停车等),辅助自动驾驶系统或驾驶员理解交警指令。
(1) 至少能够有效识别 8 种中国标准交警指挥手势(停止信号、直行信
号、左转弯信号、左转弯待转信号、右转弯信号、变道信号、减速慢行
信号、靠边停车信号)(5 分);
(2) 基于人体关键点检测(Keypoint Detection)等技术提取交警骨骼关键
点坐标,结合时序动作特征实现手势动作分类,输出手势类型及置信度
(6 分);

3. 车主手势控车(20 分)
实现基于视觉手势识别的车载人机交互系统,使车主能够通过简单的手部
手势对车辆的部分功能进行非接触式控制。系统通过车内摄像头实时捕捉车主
手部动作,识别预定义手势指令并触发对应的车辆控制操作(模拟),提供一
种更便捷、更安全的交互方式。
(1) 至少支持 6 种预定义车主手势的识别与响应,例如:手掌张开(启动
/唤醒)、握拳(确认/执行)、单指画圈(调节音量)、左右滑动(切换
功能)、拇指向上/向下(接听/挂断电话)、挥手(返回主页);支持图
片和实时视频流输入(5 分);
(2) 基于手部关键点检测(Hand Landmark Detection)实现手势特征提取
与分类,Web 端展示手部关键点骨架识别结果,包含手势识别结果、实
时追踪(5 分);
(3) 实现手势指令到车辆控制操作的映射模块,提供模拟车辆功能控制面
板(如音量调节滑块、空调温度控制、电话接听/挂断按钮等),在 Web
界面中直观展示手势控制效果(5 分);
(4) 具备手势误触发抑制机制,通过手势持续时间阈值或复合手势确认等
方式,降低误操作概率;支持手势自定义扩展,允许用户通过 MediaPipe
Model Maker 或数据采集工具添加新的手势类别数据(可选,5 分)

事件时自动分析根因并向指定渠道发送结构化告警信息,减少人工介入,提升
系统运维的自动化水平。
